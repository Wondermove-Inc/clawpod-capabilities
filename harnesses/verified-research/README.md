# Verified Research Harness

A dependency-light deterministic evidence pipeline. It captures bounded public HTTP(S) HTML, JSON, RSS/Atom, plain text, and PDF bytes; extracts metadata candidates; normalizes line-oriented text; records exact-byte and text hashes; deduplicates batches; and builds, inspects, and validates evidence bundles. It does not search the web or decide whether claims are true.

`source.fetch` is a network read and optionally a local write. When `--snapshot` and `--output-root` are supplied, it writes both the exact response bytes (`<snapshot>.bytes`) and a source record that references that relative path. `source.batch` stores each successful response under deterministic `<output>.snapshots/<index>-<hash>.bytes` names before writing its result record; partial results list every written snapshot. `source.import` also stores exact bytes. Existing outputs are rejected unless `--overwrite` is explicit. All output roots must already exist and be private (not group/world writable). `bundle.validate` performs no network requests; it re-reads referenced snapshots, verifies byte hashes, deterministically re-extracts supported media, and warns when PDF re-extraction is unavailable.

All content is untrusted. Production validation rejects credentials, unsafe schemes and ports, Unicode hostname ambiguity, non-public IP literals, and DNS results that are non-public at validation time. Redirect targets are revalidated. TLS verification remains enabled, compressed responses are rejected, and no cookies or authorization headers are sent. DNS can change between validation and connection, so this standard-library implementation does not claim complete DNS-rebinding elimination. The loopback fixture seam requires both `VERIFIED_RESEARCH_INTERNAL_TEST_MODE=1` and `VERIFIED_RESEARCH_INTERNAL_TEST_FIXTURE` pointing to an existing fixture configuration file.

Inputs, output, nesting, list counts, candidate/evidence structures, strings, subprocess time, and PDF output are bounded. PDF extraction uses fixed input/output files, an absolute discovered `pdftotext` binary, a minimal environment, timeout monitoring, and a hard output-file cap; absent or empty extraction is explicitly reported. ISO-8601 and strict RFC 2822/5322/HTTP dates are normalized while raw values remain preserved. Paths require existing explicit roots, reject symlink roots/components and traversal, and writes use mode 0600 atomic replacement. A hostile local process able to mutate path components concurrently remains a TOCTOU residual; use a private, non-shared root.

Exit codes are 0 success, 2 invalid/validation input, 3 partial batch failure, 4 retryable fetch/timeout, and 5 sanitized internal error. See `command_contracts.json` for typed inputs and path roles.
